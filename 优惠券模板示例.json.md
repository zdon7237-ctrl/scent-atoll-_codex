# 优惠券模板示例 JSON

> 说明：本文件用于给云开发数据库 `coupons` 集合提供可直接参考的模板数据。
> 使用时建议逐条手动新增，或按需要复制 JSON 内容。

---

## 1. 自动发券示例

```json
{
  "id": 1001,
  "name": "38元节日券",
  "value": 38,
  "min": 199,
  "type": "cash",
  "startTime": "2026-03-01 00:00:00",
  "endTime": "2026-12-31 23:59:59",
  "status": 0,
  "limitBrands": [],
  "desc": "全场通用节日优惠券",
  "issueMode": "auto"
}
```

适用场景：

- 节日活动券
- 用户上线自动领取

---

## 2. 内部兑换券示例

```json
{

  "min": 299,
  "type": "discount",
```

## 6. 字段提醒

每条优惠券模板建议至少包含：

- `id`：必须唯一
- `name`
- `value`
- `min`
- `type`：`cash` / `discount`
- `startTime`
- `endTime`
- `status`：模板默认写 `0`
- `limitBrands`：不限则写 `[]`
- `issueMode`：`auto` / `code`
- `redeemCode`：仅 `code` 券必填

---

## 7. 运营注意事项

### 7.1 自动券重复发放

当前版本中：

- 相同模板 `id` 的自动券不会重复发放
- 如果要再次发相同活动券，请新建模板 `id`

### 7.2 不要漏填兑换码

如果：

- `issueMode = code`
- 但没有填写 `redeemCode`

那么：

- 用户侧只能看到“兑换码无效”
- 控制台会出现配置警告

### 7.3 删除模板不会删掉用户已领券

模板删除后：

- 用户本地已领取的券仍会保留
- 直到已使用或已过期
